# 지식 구조화 도구 기획

> **원칙**: 협업/보상 이전에, 개인이 지식을 체계화하는 도구가 우수해야 한다

---

## 현황 진단

### 현재 구조
```
TopicCluster (폴더)
└── QASet (대화)
    └── Message[] (Q&A 턴)
        └── OpinionNode[] (의견)
```

### 문제점

| 문제 | 현상 | 영향 |
|------|------|------|
| **플랫함** | QASet이 서로 독립적 | 관련 지식 발견 어려움 |
| **중복** | 비슷한 질문이 여러 개 | 분산, 통합 불가 |
| **탐색 어려움** | 검색 아니면 그래프뿐 | 브라우징 불편 |
| **요약 없음** | 주제별 합성 뷰 없음 | 전체 그림 파악 불가 |
| **연결 약함** | 관계가 자동 생성뿐 | 의도적 구조화 불가 |

## 핵심: 블록 기반 지식 UI

### 철학: "Railway 스타일 블록 에디터"

모든 지식을 **블록 단위**로 다루고, 드래그/편집이 재미있어야 함

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   ┌─────────┐     ┌─────────┐     ┌─────────┐              │
│   │ 📄 Q1   │────▶│ 💡 A1   │────▶│ 📝 의견 │              │
│   │ useState │     │ 답변... │     │ 보충... │              │
│   └────┬────┘     └────┬────┘     └─────────┘              │
│        │               │                                    │
│        │          ┌────┴────┐                              │
│        │          ▼         ▼                              │
│        │     ┌─────────┐ ┌─────────┐                       │
│        │     │ 🔗 참조 │ │ ⚡ 인사│                       │
│        │     │ useReduc│ │ 이트   │                       │
│        │     └─────────┘ └─────────┘                       │
│        │                                                    │
│        └──────────────┐                                     │
│                       ▼                                     │
│                  ┌─────────┐                                │
│                  │ 📄 Q2   │                                │
│                  │ 후속질문│                                │
│                  └─────────┘                                │
│                                                             │
│   [+ 블록 추가]   [🔗 연결]   [📁 폴더로 그룹]              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 블록 타입

| 타입 | 아이콘 | 설명 | 색상 |
|------|--------|------|------|
| 질문 | 📄 | 사용자 질문 | 파랑 |
| AI 답변 | 🤖 | AI가 생성한 답변 | 회색 |
| 인간 답변 | ✍️ | 사람이 직접 작성 | 노랑 |
| 인사이트 | 💡 | 가치있는 발견 | 보라 |
| 의견 | 💬 | 보충/반박/질문 | 초록 |
| 참조 링크 | 🔗 | 다른 블록 연결 | 청록 |
| 폴더 | 📁 | 그룹핑 컨테이너 | 주황 |
| 합성 | 📋 | AI 요약 블록 | 분홍 |

### 블록 인터랙션

#### 드래그 & 드롭

```
[기존 위치]                    [드래그 중]
┌─────────┐                   ╔═════════╗
│ 📄 질문 │    ──────────▶    ║ 📄 질문 ║ ← 살짝 확대 + 그림자
└─────────┘                   ╚═════════╝
                                   │
                                   ▼
                              ┌ ─ ─ ─ ─ ┐
                              │드롭 영역 │ ← 하이라이트
                              └ ─ ─ ─ ─ ┘
```

- 블록 잡으면 살짝 확대 (scale 1.02)
- 드래그 중 원래 위치에 고스트
- 드롭 가능 영역 하이라이트
- 드롭 시 스프링 애니메이션 (framer-motion)

### 0.2 양방향 링크 (Backlinks)

**목표**: Roam/Obsidian처럼 [[링크]]로 연결, 역참조 자동

#### 동작

1. 답변 작성 시 `[[React 훅]]` 입력
2. 자동으로 해당 노드와 연결
3. 해당 노드에서 "이 노드를 참조한 곳" 표시

#### 데이터 모델

```prisma
model KnowledgeLink {
  id              String   @id @default(cuid())
  
  // 출발점 (여러 타입 가능)
  sourceNodeId    String?
  sourceNode      KnowledgeNode? @relation("LinkSource", fields: [sourceNodeId], references: [id])
  sourceMessageId String?
  sourceMessage   Message?       @relation("LinkFromMessage", fields: [sourceMessageId], references: [id])
  sourceOpinionId String?
  sourceOpinion   OpinionNode?   @relation("LinkFromOpinion", fields: [sourceOpinionId], references: [id])
  
  // 도착점
  targetNodeId    String
  targetNode      KnowledgeNode  @relation("LinkTarget", fields: [targetNodeId], references: [id])
  
  // 링크 메타
  linkText        String?  // [[링크|표시텍스트]]의 표시텍스트
  context         String?  // 링크 주변 문맥 (검색용)
  
  createdAt       DateTime @default(now())
  
  @@index([sourceNodeId])
  @@index([targetNodeId])
  @@index([sourceMessageId])
}
```

### 0.3 지식 병합 (Merge)

**목표**: 중복/유사 QASet을 하나로 통합

#### 병합 시나리오

```
[병합 전]
📄 "React 무한루프 해결" (👣 20)
📄 "useEffect 의존성 배열" (👣 35)
📄 "useEffect 무한 렌더링" (👣 15)

[병합 후]
📄 "useEffect 무한루프 해결법" (👣 70)
├── 원본 1: "React 무한루프 해결"
├── 원본 2: "useEffect 의존성 배열"
└── 원본 3: "useEffect 무한 렌더링"
```

#### 병합 로직

1. **AI 감지**: 유사 QASet 자동 추천 (임베딩 유사도)
2. **수동 선택**: 사용자가 병합 대상 선택
3. **내용 합성**: AI가 최적 답변 합성 OR 다중 답변 탭
4. **발자국 통합**: 모든 투자 합산
5. **원본 보존**: 흡수(absorbed) 상태로 남김

#### 데이터 모델 활용

```prisma
// QASet 모델 기존 필드 활용
model QASet {
  // ... 기존 필드
  
  canonicalStatus   String  @default("independent")
  // "independent" - 독립
  // "canonical" - 정식 (병합의 주체)
  // "absorbed" - 흡수됨 (병합되어 숨겨짐)
  
  canonicalParentId String? // absorbed일 때, canonical QASet 참조
}
```

#### UX: 병합 제안

```
┌──────────────────────────────────────────────────┐
│ 💡 유사한 질문이 있습니다                          │
├──────────────────────────────────────────────────┤
│                                                  │
│ 현재: "useEffect 무한루프 해결"                    │
│                                                  │
│ 유사한 기존 길:                                   │
```

## 핵심 원칙

### 1. 개인 먼저, 협업은 그 다음

```
[잘못된 순서]
협업 기능 → 지식 구조화 → 실패

[올바른 순서]
지식 구조화 (개인) → 협업 (타인과) → 경제 (인센티브)
```

### 2. Notion/Obsidian 수준의 UX

- 드래그앤드롭 정리
- 빠른 검색 (Cmd+K)
- 키보드 네비게이션
- 마크다운 + [[위키링크]]

### 3. AI는 보조, 사람이 주체

- AI가 구조 제안 → 사람이 확정
- AI가 병합 추천 → 사람이 결정
- AI가 합성 → 사람이 검토/편집

## 결론

**지식을 체계화하는 도구가 훌륭해야** 그 위에 협업과 경제가 의미있다.

현재는:
- QASet이 플랫하게 쌓임
- 관련 지식 발견이 어려움
- 중복 해결 안 됨
- 전체 그림 파악 불가

Phase 0을 먼저 완성하면:
- 주제별 깔끔한 계층 구조
- [[링크]]로 자유로운 연결
- 중복은 병합, 핵심은 합성
- 어디서든 관련 지식 즉시 참조

**"Notion for Collective AI Knowledge"**가 되어야 협업/경제가 작동한다.
